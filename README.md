## 项目介绍

学生成绩认证系统是一个专为教育机构设计的成绩管理平台，它不仅提供了传统的成绩录入与管理功能，还通过现代技术保障数据的真实性和可靠性。系统支持多角色访问控制，为管理员、教师和学生提供不同的功能和视图。

## 功能特点

* **安全可靠**: 基于区块链技术，保障数据真实性，防止成绩被篡改
* **全面记录**: 详细记录学生所有课程成绩，提供完整的学习轨迹
* **数据分析**: 智能分析成绩趋势，帮助学生了解自己的学习状况
* **权威认证**: 提供权威的成绩认证服务，方便升学和就业使用
* **多角色支持**: 为管理员、教师和学生提供不同的功能和界面
* **响应式设计**: 支持各种设备，从移动设备到桌面环境
* **简洁美观的界面**: 使用Ant Design组件，提供现代化用户体验
* **操作日志记录**: 系统记录所有用户的关键操作，方便审计和问题追踪

## 技术栈

* **前端**：
  * Next.js: React框架，支持SSR
  * Ant Design: UI组件库
  * TypeScript: 类型安全的JavaScript
* **后端**：
  * Next.js API Routes: 服务端API
  * Prisma: 数据库ORM
  * MySQL: 关系型数据库
  * JWT: 用户认证
* **部署**：
  * Vercel: 应用部署
  * MySQL: 数据库部署

## 安装与部署

### 本地开发

1. **克隆仓库**

```bash
git clone https://github.com/jwyismynane/Academic-performance-star-travel-chain2.git
cd Academic-performance-star-travel-chain2
```

2. **安装依赖**

```bash
npm install
```

3. **环境配置**

复制环境变量示例文件并根据需要修改：

```bash
cp .env.example .env
```

然后编辑.env文件，填入你的数据库连接信息（参见[数据库配置指南](#数据库配置指南)）。

4. **数据库设置**

确保MySQL已安装并运行，然后执行：

```bash
npx prisma migrate dev
npm run db:seed
```

5. **启动开发服务器**

```bash
npm run dev
```

6. **访问应用**

打开浏览器访问 http://localhost:3000

### 数据库配置指南

本系统使用MySQL数据库，下面是详细的配置步骤，特别适合数据库新手：

#### 第1步：安装MySQL数据库

1. 访问[MySQL官网](https://dev.mysql.com/downloads/mysql/)下载并安装MySQL
2. 安装过程中设置root用户的密码，请记住这个密码

#### 第2步：使用Navicat Premium创建数据库

1. 安装Navicat Premium（这是一个数据库管理工具）
2. 打开Navicat Premium并创建一个新的连接：
   * 点击左上角的"连接" → "MySQL"
   * 填写连接信息：
     * 连接名：任意名称（例如"学生成绩系统"）
     * 主机：localhost（如果MySQL在本地）
     * 端口：3306（MySQL默认端口）
     * 用户名：root（或你创建的其他用户）
     * 密码：你的MySQL密码
   * 点击"测试连接"确保连接成功
   * 点击"确定"保存连接
3. 创建新数据库：
   * 右键点击你创建的连接
   * 选择"新建数据库"
   * 数据库名输入：`student_grade_cert`
   * 字符集选择：`utf8mb4`
   * 排序规则选择：`utf8mb4_unicode_ci`
   * 点击"确定"创建数据库

#### 第3步：配置项目连接到数据库

1. 在项目根目录找到`.env`文件（如果没有，复制`.env.example`并重命名为`.env`）
2. 编辑DATABASE_URL变量：

```
DATABASE_URL="mysql://用户名:密码@主机:端口/数据库名"
```

例如，使用本地MySQL：

```
DATABASE_URL="mysql://root:你的密码@localhost:3306/student_grade_cert"
```

**注意事项：**
* 如果密码中包含特殊字符，需要进行URL编码
* 确保密码中不含有`@`符号，否则会导致连接字符串解析错误

#### 第4步：初始化数据库表

项目根目录下运行以下命令来创建数据库表：

```bash
npx prisma migrate dev
```

#### 第5步：填充初始数据

运行以下命令添加测试数据（管理员、教师、学生账号和课程）：

```bash
npm run db:seed
```

> **重要提示**：如果遇到提示找不到脚本的错误，请确保您的`package.json`文件中包含`db:seed`脚本。
> 
> 如果没有此脚本，请手动编辑`package.json`文件，在`"scripts"`部分添加：
> 
> `"db:seed": "node prisma/seed.js"`
> 
> 并安装必要的依赖：
> 
> `npm install dotenv @prisma/client bcryptjs`

* **权限问题**：确保用户有创建表的权限

### 管理员功能

管理员拥有系统的最高权限，可以：

* 管理用户：创建、编辑、删除用户
* 管理课程：创建、编辑、删除课程
* 分配教师：为课程分配教师
* 查看系统日志：监控所有用户活动
* 查看全局统计数据：系统整体使用情况
* 清空日志：清除系统操作日志记录

#### 管理员工作流程

1. 登录系统
2. 在仪表板查看系统概览
3. 创建新课程或编辑现有课程
4. 管理用户账号（添加、修改、删除）
5. 为课程分配教师
6. 查看系统日志，监控系统活动
7. 进行系统设置

* 管理课程和班级：查看负责的课程

* 生成成绩分析报告：了解学生表现

## 系统架构

系统采用三层架构：

1. **表示层**：Next.js前端应用，负责用户界面
2. **业务逻辑层**：Next.js API Routes，处理业务逻辑
3. **数据访问层**：Prisma ORM，负责数据库操作

### 目录结构

```
├── app/                     # Next.js应用主目录
│   ├── api/                 # API路由
│   ├── components/          # 共享组件
│   ├── dashboard/           # 管理仪表板页面
│   ├── login/               # 登录页面
│   └── utils/               # 工具函数
├── lib/                     # 库文件
│   ├── prisma.js            # Prisma客户端实例
│   └── logMiddleware.js     # 日志中间件
├── prisma/                  # Prisma配置
│   ├── schema.prisma        # 数据库模型定义
│   └── seed.js              # 数据库种子脚本
├── public/                  # 静态资源
└── package.json             # 项目配置
```

## 数据库说明

系统使用MySQL数据库，主要包含以下实体：

* **User**: 用户信息（管理员、教师、学生）
* **Course**: 课程信息
* **Grade**: 成绩记录
* **SystemLog**: 系统操作日志

详细的数据库结构可以在`prisma/schema.prisma`文件中查看。
